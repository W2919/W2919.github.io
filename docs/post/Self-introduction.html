Self-introduction

第一次博客作业

一、自我介绍

(1.1)在博文中简单介绍一下自己（5'）

032102345；我是小福；我的爱好是编程和打篮球；我最喜欢牛肉面（玫瑰三层）；《起风了》；保持热爱，奔赴山海。

二、思考

(2.1)回想一下你初入大学时对计算机专业的畅想 （21'）

(2.1.1)当初你是如何做出选择计算机专业的决定的？（3'）

当初选择计算机专业主要基于两个原因：一是高中时对编程的浓厚兴趣，曾在信息学竞赛中获得省级奖项；二是看到计算机技术在各行各业的广泛应用前景。家人也认为这个专业就业前景好，薪资水平较高。填报志愿时，福州大学的计算机专业在福建省内排名靠前，最终以第一志愿被录取。

(2.1.2)你认为过去两年中接触到的课程是否符合你对计算机专业的期待，为什么？（3'）

大部分课程符合期待，但也有落差。数据结构、算法分析、操作系统等核心课程内容深入，让我感受到了计算机科学的魅力。然而，部分课程如高等数学、大学物理等基础课程与计算机专业关联性不强，让我一度感到迷茫。另外，实践环节相对理论偏少，很多知识需要自己在课外项目中巩固。总体而言，专业课程体系较为完善，但教学方式和实践机会还有提升空间。

(2.1.3)你觉得计算机是你喜欢的领域吗？它是你擅长的领域吗？（6'）

计算机确实是我喜欢的领域。每当解决一个复杂的算法问题，或看到自己写的代码成功运行，都会获得巨大的成就感。特别是在开发小项目时，从需求分析到最终实现的全过程都让我充满热情。然而，是否擅长这个问题需要客观看待：在基础编程和算法思维方面，我表现较好，多次在课程设计中获得优秀；但在系统架构设计、大型项目管理方面仍有不足。与实验室的一些大神相比，我的工程能力还有差距。我认为兴趣是擅长的前提，愿意通过持续学习和实践提升自己的专业能力。

(2.1.4)将来你会选择从事计算机相关的工作吗？是的话给出你想去的城市、公司和岗位，否的话给出原因。（9'）

我计划从事计算机相关工作。首先，行业前景广阔，技术更新快，能持续保持学习状态；其次，薪资待遇相对较好，能为未来生活提供保障；最重要的是，这个领域能让我用技术解决实际问题，创造价值。

理想的发展路径是：第一选择去深圳，因为那里有腾讯、华为、大疆等顶尖科技公司，创新氛围浓厚；第二选择是杭州，阿里巴巴、网易等公司汇聚，互联网生态完善。岗位方面，我倾向于后端开发工程师或算法工程师。后端开发能让我深入理解系统架构，算法工程师则能发挥我的数学优势。短期目标是进入一线大厂积累经验，长期希望能在某个技术领域成为专家，或者创业做自己的产品。

(2.2)即将大三的你，对照前人们走过的路和描述未来发展，现在的你（12'）

(2.2.1)自我感觉你已经具备的专业知识、技能、能力有哪些？（6'）

专业知识：掌握了C/C++、Java、Python等编程语言基础；理解数据结构与算法核心概念；熟悉操作系统基本原理；了解计算机网络体系结构；具备数据库设计与SQL编写能力。

专业技能：能独立完成中小型Web应用开发（Spring Boot+Vue）；熟悉Git版本控制；掌握Linux基本操作；能使用Docker进行容器化部署；具备基本的前端开发能力（HTML/CSS/JavaScript）。

综合能力：具备良好的逻辑思维和问题分析能力；能快速学习新技术；有一定的团队协作经验（参与过3个课程项目）；具备技术文档编写能力；通过英语六级，能阅读英文技术文档。

(2.2.2)离成为一个合格的计算机专业本科毕业生，在专业知识、技能、能力上还差距哪些？（6'）

专业知识差距：分布式系统设计经验不足；对编译原理、计算机体系结构等底层知识理解不深；缺乏大规模系统性能优化经验；对新兴技术（如区块链、量子计算）了解有限。

专业技能差距：云原生技术栈（K8s、Service Mesh）实践经验不足；缺乏高并发、高可用系统设计经验；DevOps工具链使用不熟练；缺乏生产环境问题排查经验。

综合能力差距：项目管理能力不足，缺乏从0到1主导项目的经历；技术方案设计能力有待提升；沟通表达能力需要加强，特别是向非技术人员解释技术问题的能力；缺乏实际商业项目经验，对业务需求理解不够深入。

(2.3)大三是一个人生选择的十字路口，考研、工作、考公、出国，不同的选择在大三就有不同的努力方向。而无论考研还是工作的每条路径，也有许多不同的分支。（21'）

(2.3.1)根据你所了解的前辈们的经历，结合自身实际，你的选择是什么？（3'）

经过与多位学长学姐交流，结合自身情况，我选择直接就业。主要原因有三：首先，我对实践更感兴趣，希望尽早接触真实项目；其次，家庭经济条件一般，希望尽快经济独立；最后，通过实习了解到，本科毕业进入大厂的发展空间依然很大，不一定需要研究生学历。

(2.3.2)在这种选择下，你认为你相比其他同学来说有何优势，有何劣势？（9'）

优势：

  1. 项目经验丰富：大二期间参与了3个实际项目开发，包括一个电商小程序和一个人工智能课程设计项目，积累了全栈开发经验。
  2. 实习经历：暑期在本地一家软件公司实习2个月，熟悉企业开发流程和团队协作模式。
  3. 技术广度：不仅掌握后端开发，还具备前端和运维基础，能够理解整个技术栈。
  4. 学习能力强：能快速掌握新技术，曾在一周内学会Vue框架并完成项目开发。
  5. 沟通能力：担任过课程项目组长，能够有效协调团队成员，与产品、设计等部门沟通。

劣势：

  1. 学历背景：相比985高校学生，在简历筛选环节可能处于劣势。
  2. 算法能力：虽然通过了基础算法课程，但在LeetCode等平台刷题量不足，面试可能吃亏。
  3. 专业深度：对某个技术领域的钻研不够深入，缺乏核心竞争力。
  4. 大厂经历：没有在知名互联网公司实习的经历，缺乏背书。
  5. 作品集：缺乏高质量的个人作品或开源项目贡献，难以在面试中脱颖而出。
  6. 英语口语：虽然笔试成绩不错，但口语表达能力较弱，可能影响外企面试。

(2.3.3)针对你的选择，你给自己的大三设定的规划安排是什么？（9'）

第一学期（9-12月）：

  • 9-10月：系统复习数据结构与算法，每天刷2道LeetCode题目，重点攻克动态规划、图论等难点
  • 10-11月：深入学习Java并发编程和JVM原理，完成一个高并发秒杀系统项目
  • 11-12月：准备秋招，完善简历，参加校园宣讲会，争取寒假实习机会
  • 每月参加1次技术分享会或黑客马拉松，拓展人脉

寒假（1-2月）：

  • 争取进入一家互联网公司实习，积累实际项目经验
  • 利用业余时间学习云原生技术（Docker+K8s）
  • 准备英语口语，为可能的外企面试做准备

第二学期（3-6月）：

  • 3-4月：深入学习分布式系统设计，完成一个微服务项目
  • 4-5月：针对面试薄弱环节进行强化训练，模拟面试
  • 5-6月：参加春季招聘，争取正式工作offer
  • 持续维护个人技术博客，每月至少2篇技术文章

贯穿全年：

  • 每周投入10小时学习新技术
  • 每月阅读1本技术书籍
  • 每季度参与1个开源项目贡献
  • 保持健身习惯，每周3次运动，确保身心健康

(2.4)了解最近的热点事件吗？比如日本核废水排海计划等；对中国长远经济的发展。对此你有什么看法和感触吗？（6'）

我持续关注日本核废水排海事件。从技术角度看，ALPS处理系统确实能去除大部分放射性物质，但氚的处理仍是世界难题。日本政府的决策过程缺乏透明度，未充分考虑邻国和国际社会的关切。这提醒我们：技术发展必须考虑伦理责任，环境保护不应让位于经济利益。

对中国长远经济发展，我认为有几个关键点：首先，人口老龄化和少子化趋势将对经济增长模式提出挑战，需要从人口红利转向技术红利；其次，中美科技竞争背景下，核心技术自主可控变得尤为重要，特别是在芯片、操作系统等基础领域；最后，绿色低碳转型是必然趋势，新能源、数字经济将成为新增长点。作为计算机专业学生，我深感责任重大——我们这一代人需要在关键技术领域实现突破，为国家高质量发展贡献力量。

三、未来的期望

(3.1)你愿意为了一个你感兴趣的、或得到实战锻炼的项目熬夜吗？你想脱离小菜鸟的level吗？如何评价上上上、上上、上一届同学说的"真正和我的队友们一起见识了凌晨四点的福大。一起熬夜，一起吐槽，一起说说笑笑，一起打代码。一切一切的都是'美好'的回忆。"（5'）

我完全愿意为感兴趣的项目熬夜。大二做课程设计时，曾和队友连续三天熬夜到凌晨三点，虽然辛苦，但看到项目成功运行的那一刻，所有疲惫都化为成就感。我认为，真正的成长往往发生在舒适区之外，而熬夜赶项目正是突破自我的过程。

我非常渴望脱离"小菜鸟"的level。在技术领域，永远有更优秀的同行值得学习，保持"菜鸟心态"很重要，但也要不断精进技能，从"知道怎么做"到"知道为什么这么做"，最终达到"能设计怎么做"的层次。

对前辈们的那句话，我深有共鸣。那些共同奋斗的夜晚，不仅锻炼了技术能力，更培养了团队默契和友谊。在压力下互相支持、共同解决问题的经历，往往比技术本身更珍贵。这些回忆之所以"美好"，是因为它们见证了我们的成长和蜕变。我相信，在软工实践中，我也会收获这样的美好回忆。

针对上述问题(1)、(2)、(3)，你对这门课的期待是什么？你打算平均每周拿出多少个小时用在这门课上，以达成你的期待以及你在(1)或(2)或(3)上的目标？（10'）

我对软工实践课程充满期待。首先，希望通过真实项目锻炼工程能力，将理论知识转化为实践技能；其次，期待在团队协作中学习沟通协调，体验软件开发全流程；最后，希望接触前沿技术，为就业做好准备。

为了达成这些目标，我计划平均每周投入12小时在这门课上：

  • 4小时用于课堂学习和讨论
  • 6小时用于项目开发和团队协作
  • 2小时用于技术学习和文档编写

在时间安排上，我会优先保证项目进度，特别是关键节点前会适当增加投入。同时，注重平衡学习与休息，避免过度疲劳影响效率。我相信，通过这门课的严格训练，我能在软件工程能力上实现质的飞跃，为即将到来的就业季打下坚实基础。
