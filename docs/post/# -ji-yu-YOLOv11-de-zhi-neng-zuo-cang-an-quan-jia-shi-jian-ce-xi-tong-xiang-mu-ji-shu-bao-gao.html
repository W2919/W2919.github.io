# 基于YOLOv11的智能座舱安全驾驶检测系统项目技术报告

一、开发背景、应用价值及潜在客户群

（一）开发背景

随着汽车行业的快速发展和人们对驾驶体验要求的不断提高，单一功能的车载设备已难以满足用户需求。当前车载系统存在功能分散、操作繁琐、驾驶辅助不足等问题，例如导航、多媒体播放、车辆周边环境监测等功能往往相互独立，给驾驶员带来诸多不便。

本项目旨在开发一款整合了多种功能的车载多媒体与辅助驾驶系统，通过融合计算机视觉、导航定位、数据库管理等技术，解决驾驶过程中信息获取不及时、操作复杂、驾驶安全性有待提升等关键问题，为用户提供一站式的车载服务体验。

（二）应用价值

  1. 提升驾驶安全性：通过基于YOLOv11模型的道路车辆检测功能，实时监测车辆周边环境，及时向驾驶员发出提醒，减少交通事故的发生。
  2. 增强驾驶便利性：整合导航、音乐播放、照片视频管理等功能，让驾驶员在驾驶过程中能够便捷地操作各项功能，无需频繁切换设备。
  3. 个性化服务体验：支持用户注册登录、信息管理等功能，根据用户的喜好和习惯提供个性化的服务，如个性化音乐推荐、常用路线记录等。

（三）潜在客户群

  1. 汽车制造商：可将该系统作为车载预装系统，提升汽车的智能化水平和市场竞争力。
  2. 汽车后市场服务商：为已出厂的车辆提供系统加装服务，满足车主对车载系统升级的需求。
  3. 个人车主：尤其是对车载多媒体功能和驾驶辅助功能有较高需求的车主，该系统能为其提供更优质的驾驶体验。

二、系统设计方案

（一）E-R设计图

系统的主要实体包括用户（User）、音乐（Music）、照片（Photo）、视频（Video）等，各实体之间的关系如下：

  • 用户（User）与音乐（Music）：一对多关系，一个用户可以拥有多个音乐文件。
  • 用户（User）与照片（Photo）：一对多关系，一个用户可以拥有多个照片文件。
  • 用户（User）与视频（Video）：一对多关系，一个用户可以拥有多个视频文件。
    各实体的属性如下：
  • User：ID、username、password、phone_number、user_img、user_road_video_path等。
  • Music：ID、name、file_path、UID（关联User的ID）等。
  • Photo：ID、name、file_path、UID（关联User的ID）等。
  • Video：ID、name、file_path、UID（关联User的ID）等。

（二）系统架构图

系统采用分层架构设计，从上到下依次为：

  1. 表现层：基于PyQt5实现的用户界面，包括主窗口、登录窗口、各个功能模块窗口等，负责与用户进行交互。
  2. 业务逻辑层：包含各类控制器和功能实现类，如LoginController、RegisterController、MusicManager、PhotoManager等，处理业务逻辑，协调各层之间的通信。
  3. 数据访问层：通过DatabaseManager实现对数据库的操作，包括数据的增删改查等，为业务逻辑层提供数据支持。
  4. 外部服务层：对接高德地图API、天气API等外部服务，为系统提供导航、天气查询等功能。

（三）功能模块图

系统主要包含以下功能模块：

  1. 用户管理模块：负责用户的注册、登录、信息修改等功能。
  2. 主控制模块：协调各个功能模块的切换和运行，是系统的核心控制部分。
  3. 驾驶辅助模块：基于YOLOv11模型实现道路车辆检测、视频录制等功能。
  4. 多媒体模块：包括音乐播放、照片管理、视频管理等功能。
  5. 导航模块：基于高德地图API实现地图显示、路径规划、定位等功能。
  6. 天气查询模块：获取并显示当前城市的天气信息。

（四）功能介绍

  1. 用户管理模块

    • 注册：支持用户通过手机号和密码进行注册，需验证验证码。
    • 登录：提供密码登录和手机短信登录两种方式。
    • 信息修改：允许用户修改密码、绑定新号码、上传头像等。

  2. 主控制模块

    • 功能切换：通过点击不同的按钮，切换到对应的功能模块窗口。
    • 数据更新：定时更新时间、天气等信息。

  3. 驾驶辅助模块

    • 视频录制：可实时录制车辆前方的视频，并保存到系统中。
    • 车辆检测：利用YOLOv11模型对视频流中的车辆进行检测，并在界面上显示检测结果和提醒信息。

  4. 多媒体模块

    • 音乐播放：支持加载本地音乐文件，实现播放、暂停、切换等控制，展示音乐列表。
    • 照片管理：支持多线程加载照片，可对照片进行添加、删除、查看等操作，展示照片列表。
    • 视频管理：支持多线程加载视频，实现视频的播放、倍速播放、日期筛选、删除等功能。

  5. 导航模块

    • 地图显示：加载高德地图，显示实时路况。
    • 路径规划：根据起点和终点规划最佳行驶路径。
    • 定位功能：获取当前车辆的位置信息并在地图上显示。

  6. 天气查询模块

    • 天气获取：通过调用天气API获取当前城市的天气信息，包括白天和夜间的天气状况、温度、风向等。
    • 天气显示：根据当前时间（白天或夜间）显示对应的天气信息。

三、技术开发方案

（一）使用的框架/库

  1. PyQt5：用于开发图形用户界面，实现窗口、按钮、标签等界面元素的创建和交互。
  2. ultralytics YOLO：提供YOLOv11模型，用于实现道路车辆检测功能。
  3. requests：用于发送网络请求，获取天气信息、高德地图API数据等。
  4. OpenCV（cv2）：用于处理视频流，包括视频的读取、录制、帧处理等。
  5. PyQt5.QtWebEngineWidgets：用于在界面中嵌入网页，显示高德地图。
  6. json：用于解析和处理配置文件、API返回的JSON数据等。
  7. tempfile：用于创建临时文件，存储地图HTML等临时数据。

（二）编程语言

项目主要使用Python编程语言进行开发，Python具有丰富的库和框架，便于快速开发和集成各种功能。

（三）编程工具

  1. PyCharm：作为主要的集成开发环境（IDE），提供代码编辑、调试、版本控制等功能，提高开发效率。
  2. Qt Designer：用于设计PyQt5的用户界面，可直观地拖拽界面元素，生成.ui文件，再通过PyQt5转换为Python代码。

（四）设计工具

  1. Visio：用于绘制E-R图、系统架构图、功能模块图等，清晰展示系统的设计结构。

（五）系统环境

  1. 操作系统（OS）：Windows 10及以上版本。
  2. 硬件需求：
    • 处理器：Intel Core i5及以上或同等性能的处理器。
    • 内存：至少4GB RAM，推荐8GB及以上。
    • 硬盘：至少500MB可用空间，用于安装软件和存储数据。
    • 摄像头：用于采集视频流，实现车辆检测和视频录制功能。
    • 网络适配器：用于连接互联网，获取天气信息、地图数据等。

四、系统安装与部署步骤

  1. 安装Python

    • 访问Python官方网站（https://www.python.org/），下载Python 3.8及以上版本的安装包。
    • 运行安装包，勾选“Add Python to PATH”选项，然后点击“Install Now”进行安装。
    • 安装完成后，打开命令提示符（CMD），输入“python --version”，若显示Python版本信息，则安装成功。

  2. 安装所需库

    • 打开CMD，依次输入以下命令安装所需的库： 
      pip install PyQt5
pip install ultralytics
pip install requests
pip install opencv-python
pip install PyQtWebEngine

  3. 配置数据库

    • 本项目使用的数据库相关操作已封装在DatabaseManager中，无需额外安装数据库服务，系统会自动处理数据库文件。

  4. 获取相关API密钥

    • 高德地图API：访问高德开放平台（https://lbs.amap.com/），注册账号并创建应用，获取API密钥，将密钥替换到weather.py和RealTimeMapApp类中对应的位置。
    • 天气API：若使用的天气API需要密钥，按照相应平台的要求获取并配置密钥。

  5. 准备模型文件

    • 将YOLOv11模型文件（best.pt、yolo11n-seg.pt）放置在项目根目录下。

  6. 配置文件设置

    • 检查config.json文件中的各项路径配置，确保图标、视频、照片等文件的存储路径正确。

  7. 运行程序

    • 下载项目代码到本地目录，解压后进入项目根目录。
    • 找到主程序文件（如appMainWin.py），在CMD中输入“python appMainWin.py”，运行程序。

五、系统测试方法

（一）测试用例设计

针对系统的各个功能模块，设计如下测试用例：

  1. 用户管理模块测试

    • 注册功能：输入未注册的用户名、密码、手机号等信息，验证是否能成功注册；输入已注册的用户名或手机号，验证是否提示相应错误信息。
    • 登录功能：使用正确的用户名和密码（或手机号和验证码）登录，验证是否能成功登录；使用错误的信息登录，验证是否提示相应错误信息。
    • 信息修改功能：修改密码、绑定新号码、上传头像等，验证修改后信息是否正确更新。

  2. 驾驶辅助模块测试

    • 视频录制功能：点击录制按钮，验证是否能正常录制视频；点击停止按钮，验证视频是否能正确保存到指定路径，且数据库中是否有相应记录。
    • 车辆检测功能：打开摄像头，验证是否能实时检测到车辆，并在界面上正确显示检测结果和提醒信息。

  3. 多媒体模块测试

    • 音乐播放功能：添加本地音乐文件，验证是否能正常播放、暂停、切换歌曲，音乐列表是否正确显示。
    • 照片管理功能：添加照片，验证是否能正常显示；删除照片，验证照片是否从列表和存储路径中移除。
    • 视频管理功能：添加视频，验证是否能正常播放、倍速播放；按日期筛选视频，验证筛选结果是否正确；删除视频，验证视频是否从列表和存储路径中移除。

  4. 导航模块测试

    • 地图显示功能：验证地图是否能正常加载和显示，是否能显示实时路况。
    • 路径规划功能：输入起点和终点，验证是否能规划出合理的路径。
    • 定位功能：验证是否能正确获取当前位置并在地图上显示。

  5. 天气查询模块测试

    • 验证是否能正确获取当前城市的天气信息，且能根据白天和夜间切换显示对应的天气内容。
